# 文档更新规范

## 文档同步规则 ⚠️

当需求或架构变更时，必须同步更新所有相关文档：
- `docs/requirements.md` - 需求规范
- `docs/design.md` - 系统设计
- `docs/tasks.md` - 任务清单

## 文档更新哲学

更新文档时要像从头创建文档一样写作：
- 文档应显得连贯完整
- 不显示增量更新的痕迹
- 保持语调、结构和技术深度的一致性

## 版本信息

每次更新文档时，更新文档末尾的版本信息：
```markdown
---
*本文档版本：1.0* (保持1.0)
*更新日期：YYYY-MM-DD*
*作者：JN.L*
```

## 任务状态标记

tasks.md 中使用以下状态标记：
- ⬜ 未开始
- 🟦 进行中
- ✅ 已完成
- ⏸️ 暂停
- ❌ 取消
